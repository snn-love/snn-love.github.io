about_拷贝构造

发表于 2021-11-25

一些小的点：

    #### 什么时候会出现拷贝构造？

        使用一个对象初始化另一个对象时。

        使用 函数的参数为对象本身时。

        当函数返回为一个对象时。

    #### 为何要使用拷贝构造？

        可以不使用，那么编译器会自动给你生成一个默认拷贝构造。创建使用时，可以自己决定那些东西拷贝。

    #### 拷贝构造是怎样进行拷贝构造？

        概念：拷贝构造是成员级别的拷贝，

        简单描述下：int型拷贝给int型，char型拷贝给char型，如果是一个对象，那么会让那个对象的类的拷贝构造来拷贝构造这个对象。

    #### 如果成员变量有指针，此时的拷贝构造是怎么拷贝的？

        同上，会发生指针的拷贝。也就是说拷贝源指针和拷贝后的指针，这两个指针指向了同一片内存。

        说到了指针就不得不说引用。发生的是两个引用捆绑着同一个变量。

它是个啥？

    不是个真实指针，当创建一个自定义类型的对象时（class a{….}；int main(){a A;}） A就是那个被’This’所指的对象。每当使用这个对象的成员函数，哪个函数的参数表就会多出一个隐藏的参数,就是哪个this指针。

刨根溯源

“This”指针到底是个啥？

它是成员函数和成员变量关联的桥梁。

    成员函数最终被编译成与对象无关的普通函数，除了成员变量，会丢失所有信息，所以编译时要在成员函数中添加一个额外的参数，把当前对象的首地址传入，以此来关联成员函数和成员变量。

                                            —-《 C++函数编译原理和成员函数的实现 》

关于this指针的一个精典回答:

当你进入一个房子后，
你可以看见桌子、椅子、地板等，
但是房子你是看不到全貌了。
对于一个类的实例来说，
你可以看到它的成员函数、成员变量，
但是实例本身呢？
this是一个指针，它时时刻刻指向你这个实例本身。

发表于 2021-10-06

约束：权限为 root ，phpmyadmin管理页面泄露

原理：全局函数的可控，导致日志属性（路径，内容）的可控。

  • general log 指的是日志保存状态，一共有两个值（ON/OFF）ON代表开启 OFF代表关闭。

  • general log file 指的是日志的保存路径。

set general_log =’on’; 状态设置为打开。默认为关闭

set global general_log_file=’E:/oldphpstday/WWW/shell.php’;

将路径设置为网站目录，并命名为shell.php
