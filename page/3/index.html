Each Test Case covers several OWASP tests which also is useful for the report document. I’ve also added 15 extra Tests Cases marked by the EXTRA-TEST. I hope it will be useful in both penetration test projects and bug-bounty.

一些小的点：

    #### 什么时候会出现拷贝构造？

        使用一个对象初始化另一个对象时。

        使用 函数的参数为对象本身时。

        当函数返回为一个对象时。

    #### 为何要使用拷贝构造？

        可以不使用，那么编译器会自动给你生成一个默认拷贝构造。创建使用时，可以自己决定那些东西拷贝。

    #### 拷贝构造是怎样进行拷贝构造？

        概念：拷贝构造是成员级别的拷贝，

        简单描述下：int型拷贝给int型，char型拷贝给char型，如果是一个对象，那么会让那个对象的类的拷贝构造来拷贝构造这个对象。

    #### 如果成员变量有指针，此时的拷贝构造是怎么拷贝的？

        同上，会发生指针的拷贝。也就是说拷贝源指针和拷贝后的指针，这两个指针指向了同一片内存。

        说到了指针就不得不说引用。发生的是两个引用捆绑着同一个变量。

它是个啥？

    不是个真实指针，当创建一个自定义类型的对象时（class a{….}；int main(){a A;}） A就是那个被’This’所指的对象。每当使用这个对象的成员函数，哪个函数的参数表就会多出一个隐藏的参数,就是哪个this指针。

它有什么用？

    先放代码

编译器是如何判断出 a 的还是 b的指针的？就是”This”指针发挥的作用 。

它如何用？

两种用法：
1,在定义成员函数时，当函数需要返回 对象本身时。

2,当函数参数与类成员函数名字冲突时可以（this→i=i）来区别于传入的”i”

刨根溯源

“This”指针到底是个啥？

它是成员函数和成员变量关联的桥梁。

    成员函数最终被编译成与对象无关的普通函数，除了成员变量，会丢失所有信息，所以编译时要在成员函数中添加一个额外的参数，把当前对象的首地址传入，以此来关联成员函数和成员变量。

                                            —-《 C++函数编译原理和成员函数的实现 》

关于this指针的一个精典回答:

当你进入一个房子后，
你可以看见桌子、椅子、地板等，
但是房子你是看不到全貌了。
对于一个类的实例来说，
你可以看到它的成员函数、成员变量，
但是实例本身呢？
this是一个指针，它时时刻刻指向你这个实例本身。

end

设计员就想啊。。。。干脆就给C++加个面向对象的能力。在给C++来个翻译成C的功能（C++到C的翻译器）。不就O了~。偷懒总会有BUG，为了解决 成员变量于全局变量的区别，所以才引入了”This”指针。
